## Где менять данные

**Всё меняется в `data.js`, объект `APP_DATA`:**

| Что менять | Где в data.js |
|---|---|
| Имя юриста, должность, биография | `lawyer.name`, `lawyer.title`, `lawyer.bio` |
| Статистика юриста (дела, сумма, %) | `lawyer.stats` |
| Реквизиты (ИНН, ОГРН, адрес) | `company` |
| Телефон для кнопок звонка | `contact.phone` + `contact.phoneDisplay` |
| Username Telegram-бота | `contact.telegram` |
| URL бэкенда для приёма заявок | `contact.webhookUrl` |
| Ссылки политики и оферты | `contact.privacyUrl`, `contact.offerUrl` |
| Вопросы и варианты квиза | `quiz.steps` |
| Результаты квиза (тексты) | `quiz.results.mfc`, `.court`, `.individual` |
| Услуги (название, цена, описание) | `services` — массив объектов |
| Обязательный дисклеймер | `legalWarning` |

## Логика квиза (результат)

Логика вычисления в `app.js`, метод `screens.result.calculate()`:

- **MFC (бесплатное банкротство):** долг ≤ 1 млн + нет ипотеки + нет официального дохода + просрочка > 3 мес
- **Court (судебное банкротство):** долг 300к–1млн+ + просрочка ≤ 6 мес или > 3 мес
- **Individual (индивидуальный анализ):** все остальные случаи

## Подключение реального бэкенда

В `app.js`, метод `screens.form.submit()` — найдите блок с комментарием:
```
// ─── ОТПРАВКА НА БЭКЕНД ─────────────────────────────────
```

Замените `setTimeout(onSuccess, 1500)` на реальный `fetch()`.

Данные, которые отправляются:
```json
{
  "name": "Имя пользователя",
  "phone": "+79991234567",
  "time": "morning | afternoon | evening",
  "source": "home | result | service",
  "quizAnswers": { "debt": "medium", "housing": "own", ... },
  "userId": 123456789,
  "username": "telegram_username"
}
```

## Тестирование в браузере

1. Откройте `tg-app/index.html` в браузере (через Live Server или VS Code)
2. Появятся mock-кнопки: синяя «Далее» снизу + серая «← Назад» сверху
3. Квиз, форма, все экраны — полностью работают

**В браузере отключены:**
- `requestContact()` — вместо этого появляется поле ввода телефона
- `HapticFeedback` — игнорируется без ошибок
- `CloudStorage` — заменён на `localStorage`
- Тема Telegram — используются CSS-фолбеки (светлая тема)

## Публикация в Telegram

1. Создайте бота через @BotFather → `/newbot`
2. Настройте Mini App: `/newapp` в BotFather
3. Укажите URL вашего HTTPS-хостинга (не localhost)
4. Вставьте `contact.telegram` в data.js
5. Реализуйте бэкенд для приёма заявок (Python FastAPI / Node.js)
6. Вставьте URL бэкенда в `contact.webhookUrl`

## Что менять в index.html

`index.html` менять почти не нужно — весь контент рендерится через JS.

Исключения:
- Экран `#screen-form` — поля формы, если нужно добавить новые
- Экран `#screen-success` — текст после отправки
- Экран `#screen-welcome` — факты на карточках (или перенести в data.js)

## Юридические требования (не удалять!)

По ст. 28.1 38-ФЗ (с 01.01.2026):
- Дисклеймер в `APP_DATA.legalWarning` обязателен на сайте
- Нельзя писать «гарантируем», «100%», «спишем за N дней»
- Нужна политика конфиденциальности (ссылка в форме и в «О нас»)
- Реквизиты организации обязательны (экран «О нас»)
- Согласие на ПДн в форме — обязательно
