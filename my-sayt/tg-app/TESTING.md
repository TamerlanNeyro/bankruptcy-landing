## 3. Локальный запуск (браузер)

**Установка не нужна.** Сборки нет. Зависимостей нет.

1. Открыть `tg-app/index.html` в любом браузере  
   (рекомендуется через Live Server в VS Code, чтобы работали относительные пути)
2. Появятся mock-кнопки:
   - Синяя кнопка внизу — эмулирует `MainButton` Telegram
   - Серая кнопка `← Назад` вверху — эмулирует `BackButton` Telegram
3. Все экраны доступны, квиз и форма работают полностью

**Как браузер отличается от Telegram:**

| Функция | В браузере | В Telegram |
|---|---|---|
| `inTelegram` | `false` (initData = "") | `true` |
| `requestContact()` | показывает поле ввода | открывает диалог TG |
| `CloudStorage` | `localStorage` с префиксом `tma_` | нативное хранилище TG |
| `HapticFeedback` | игнорируется | вибрация устройства |
| Тема | CSS-фолбеки (светлая) | цвета из темы TG |
| Имя пользователя | не подставляется | `tg.initDataUnsafe.user.first_name` |

## 4. Переменные окружения

Файл `.env` (не коммитится):
```
BOT_TOKEN=<токен бота из BotFather>
```

В Vercel добавлен вручную через `vercel env add BOT_TOKEN production`.  
Без `BOT_TOKEN` webhook вернёт 200 но не ответит на сообщения бота.

## 5. Маршрут пользователя (data flow)

```
Пользователь → /start в боте
  → webhook.js отправляет приветствие + reply keyboard (2 кнопки)
  → пользователь нажимает «Открыть приложение»
  → Telegram открывает https://tg-app-six-wine.vercel.app

index.html загружается →
  Splash (1.5 сек)
    ├── quiz_done в CloudStorage = true  → Home (с таббаром)
    └── иначе                           → Welcome

Welcome → [Main Button] → Quiz (5 шагов) → Result → Form → Success

Таббар (Home / Диагностика / Услуги / Обо мне / Контакты)
  доступен только после первого прохождения квиза
```

## 7. Тестирование бота (webhook)

### Проверить что webhook установлен
```bash
curl "https://api.telegram.org/bot<BOT_TOKEN>/getWebhookInfo"
```
Ожидается: `"url": "https://tg-app-six-wine.vercel.app/api/webhook"`

### Имитировать /start напрямую
```bash
curl -X POST "https://tg-app-six-wine.vercel.app/api/webhook" \
  -H "Content-Type: application/json" \
  -d '{"message":{"text":"/start","chat":{"id":123},"from":{"first_name":"Тест"}}}'
```
Ожидается:
```json
{
  "method": "sendMessage",
  "chat_id": 123,
  "reply_markup": {
    "keyboard": [
      [{"text": "🚀 Открыть приложение", "web_app": {...}}],
      [{"text": "📋 Записаться на консультацию", "web_app": {...}}]
    ],
    "persistent": true
  }
}
```

### Имитировать /help
```bash
curl -X POST "https://tg-app-six-wine.vercel.app/api/webhook" \
  -H "Content-Type: application/json" \
  -d '{"message":{"text":"/help","chat":{"id":123},"from":{}}}'
```
Ожидается: `method: "sendMessage"` с inline_keyboard (одна кнопка)

### Несуществующая команда
```bash
curl -X POST "https://tg-app-six-wine.vercel.app/api/webhook" \
  -H "Content-Type: application/json" \
  -d '{"message":{"text":"привет","chat":{"id":123},"from":{}}}'
```
Ожидается: `{"ok": true}` — тихий ответ

### GET-запрос (health check)
```bash
curl "https://tg-app-six-wine.vercel.app/api/webhook"
```
Ожидается: `{"ok": true}` — 200 OK

```css
--navy:       #0C243D   /* тёмно-синий — фон карточки адвоката, hero-блок */
```
